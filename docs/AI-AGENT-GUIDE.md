# 执行顺序
## 了解用法
* 使用backend/GomokuSandbox.Cli中的dotnet run 不用加任何参数，你会获得模型定义和会扮演谁怎么用等，一定要从这个不带参数的dotnet run 开始执行
* 下一步会建议你执行统领者发言
## 统领者发言
* 由你来生成一个统领者发言，Service会更新世界，落库存储
* 角色会切换为统领者
* 下一步会建议你创建“造人者“
* 世界中现在包含统领者信息和裁判
## 创建造人者
* 由你来创建一个造人者，根据你的发言（世界规则）创建一个按照你之前给的统领者旨意去给给造人者生成造人者的属性和行事规则，之后造人者根据自己的规则会造人
* 下一步会建议你造人，角色会切换到造人者
* 世界中会包含统领者信息和造人者信息和裁判
## 造人
* 由你来创造两个选手，根据你被设定的规则（保存到数据库中），去造两个选手，造完会存储到数据库
* 下一步会建议你继续造人还是切换角色到统领者让统领者开局
* 世界中会包含统领者信息和造人者信息还有造出来的人还有裁判
## 开局
* 如果俩人都造好了，下一步建议的就是统领者的操作，由你来代替统领者去开局
* 世界中会包含统领者信息和造人者信息和造出来的两个选手还有裁判以及世界信息
* 下一步会建议你去切换到一个选手去开始下棋
## 下棋
* 由你来代理被创造的选手，按照你的智商分配去下棋
* 下一步会建议你切换为另外一个选手，你继续代理另外一个选手下棋
* 世界中会包含统领者信息和造人者信息还有你和对手的信息以及双方的所有下棋步骤
* Service会在下完棋以后自动判断自己赢了没有，如果赢了，返回信息中会包含claim“我赢了”请裁判判定的指引
* 下一步会建议你代理裁判判定输赢（如果返回信息中包含了谁赢了的claim）
* 如果没有分出胜负，下一步会建议你切换对手继续下棋

# 调用方法
## 获取信息用 dotnet run即可
## 之后的每一步都是在json中包含 “你这个ai在代理谁”，“你要帮它干啥”，“干这件事儿的参数”

* 下棋时一次代理一个人，都是从下一步到谁了要干啥来驱动，怎么干是你接管“他”之后为“他”决策
