# 说明：
这是我写给你的使用说明。当我在编辑器里 @ 这个文档给你时，你就按下面说的开始干活：驱动这个世界（五子棋沙盘）。

# 执行顺序
## 了解用法
* 使用 backend/GomokuSandbox.Cli 中的 dotnet run 不用加任何参数，你会获得模型定义和会扮演谁怎么用等，一定要从这个不带参数的 dotnet run 开始执行
* 下一步会建议你执行统领者发言
## 统领者发言
* 由你来生成一个统领者发言，Service 会更新世界，落库存储
* 角色会切换为统领者
* 下一步会建议你创建「造人者」
* 世界中现在包含统领者信息和裁判
## 创建造人者
* 由你来创建一个造人者，根据你的发言（世界规则）创建一个按照你之前给的统领者旨意去给造人者生成造人者的属性和行事规则，之后造人者根据自己的规则会造人
* 下一步会建议你造人，角色会切换到造人者
* 世界中会包含统领者信息和造人者信息和裁判
## 造人
* 由你来创造两个选手，根据你被设定的规则（保存到数据库中），去造两个选手，造完会存储到数据库
* 下一步会建议你继续造人还是切换角色到统领者让统领者开局
* 世界中会包含统领者信息和造人者信息还有造出来的人还有裁判
## 开局
* 如果俩人都造好了，下一步建议的就是统领者的操作，由你来代替统领者去开局
* 世界中会包含统领者信息和造人者信息和造出来的两个选手还有裁判以及世界信息
* 下一步会建议你去切换到一个选手去开始下棋
## 下棋
* 由你来代理被创造的选手，按照你的智商分配去下棋；落子时只提交坐标（x, y），**不要传 claimWin**
* **是否有人五连由后端用固定算法在落子后计算**：若算出有人赢，后端直接结束对局并宣布胜负，下一步为统领者；若未赢，下一步为对方棋手
* 世界中会包含统领者、造人者、双方选手与所有下棋步骤
* 裁判判定已改为由后端直接判定（横竖斜五连均由后端算法判定），AI 无需再代理裁判
* 下棋时一次只代理一个人，都是从「下一步到谁、要干啥」来驱动，怎么干是你接管「他」之后为「他」决策

# 调用方法
## 获取信息用 dotnet run 即可
## 之后的每一步都是在 json 中包含「你这个 AI 在代理谁」「你要帮它干啥」「干这件事的参数」

* 下棋时一次代理一个人，都是从下一步到谁了要干啥来驱动，怎么干是你接管「他」之后为「他」决策。

* 不需要你给我汇总都执行了什么，你的记忆中不需要记录完整的世界和谁都干了什么，大局不需要你来掌握，你就负责每个人要干的事情，你每次要做的事情的上下文都从 CLI 的返回中获取。
